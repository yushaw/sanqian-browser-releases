## Permission Justifications

**For Chrome Web Store submission - explain each sensitive permission:**

### 1. Debugger Permission
```
Justification:
We use the Chrome DevTools Protocol (CDP) via Puppeteer to enable core web automation features. This permission is essential for:
- Reading webpage DOM structure for AI task analysis
- Simulating user interactions (clicks, typing, scrolling)
- Navigating between pages automatically
- Extracting content to fulfill user-requested tasks
- Capturing page state for AI planning

All automation is user-initiated through our chat interface and runs locally in the user's browser. We do not collect or transmit any browsing data.
```

### 2. `<all_urls>` Host Permission
```
Justification:
This permission allows the extension to inject content scripts and interact with any website the user wants to automate. Users explicitly choose which websites to interact with through natural language task requests. We need broad host permissions because:
- Users may want to automate tasks on any website
- AI agents need to read page content to understand and execute tasks
- Content scripts must be injectable for DOM manipulation

We do not access or interact with websites without explicit user instruction.
```

### 3. Tabs Permission
```
Justification:
Required for managing browser tabs during automation tasks:
- Opening new tabs when users request navigation to new pages
- Switching between tabs for multi-page workflows
- Closing tabs after task completion
- Getting tab information (URL, title) for context

Tab management only occurs during active user-initiated automation tasks.
```

### 4. Web Navigation Permission
```
Justification:
Required to monitor page load events and ensure proper automation timing:
- Detecting when pages finish loading before interaction
- Waiting for stable DOM state before executing actions
- Handling dynamic content that loads after initial page render

This ensures reliable automation execution and prevents errors from interacting with partially loaded pages.
```
